Gravity

It's a "Two Dimensional Arcade Multiplayer Orbital Space Shooter". Two friends and I made this as our project for CS 241's honors section using C++ and SDL. All objects in the system are affected by gravity, which makes things interesting because bullets don't go in straight lines. You control a small spaceship and pilot it through an accurately simulated solar system while trying to shoot other spaceships. I did all of the physics, while networking and graphics were taken care of by my teammates. It's a game at heart, but in theory you could tweak some parameters and get a fairly accurate simulation of the solar system. The code is not as embarrassing as Rougue's, but of course corners get cut when deadlines approach. Here's a link to the github; the repository also has a poster we made with more information.

Rougue

A game made the first semester of my freshman year, in collaboration with eleven other people. We used the Unity Engine, C# as the language of choice, and git as version control. I came up with the idea and handled the overall architecture. The game is a real-time rogue-like, featuring procedural dungeon generation, levels, items, spells, and perma-death. It was a lesson on how not to do many things. It was among the most complete projects in the class, though.
